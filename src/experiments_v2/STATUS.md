# Experiments V2 Status

`experiments_v2` - это текущий основной исследовательский контур проекта.

## Роль папки

Здесь должны находиться:

- новые гипотезы;
- текущие baseline-эксперименты;
- сравнение таргетов, фичей и архитектур;
- артефакты, которые помогают понять, какой подход сейчас лучший.

## Что считать источником истины внутри `experiments_v2`

- `run.py` в папке конкретного эксперимента - точка входа
- `metrics.json` - краткий результат эксперимента
- `predictions.csv` - детальный вывод при наличии
- `INDEX.md` - фактический индекс директорий и артефактов
- `REGISTRY.md` - короткая карта completed / active / deferred
- `EXPERIMENTS.md` - широкий журнал/реестр гипотез и результатов

## Практическое правило

Если появляется новая исследовательская идея, по умолчанию она должна добавляться сюда, а не в legacy pipeline и не в `web/`.

## Границы ответственности

`experiments_v2` не обязан:

- быть полностью совместимым со старым `run_pipeline.py`;
- соответствовать demo web по структуре данных и моделям;
- сохранять обратную совместимость со всеми старыми артефактами.

## Что полезно поддерживать в актуальном состоянии

- краткое описание сильного baseline;
- список лучших завершенных экспериментов;
- явное понимание, какой эксперимент сейчас считать лидирующим кандидатом.

Если потребуется, следующий шаг - сделать компактный реестр completed / active / abandoned поверх существующего `EXPERIMENTS.md`.
