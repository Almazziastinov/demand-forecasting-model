# Веб-приложение прогнозирования спроса

## Что сделано

### 1. Архитектура
- **Flask-приложение** (`web/app.py`) — REST API для прогнозирования
- **Обработка данных** (`web/data_processing.py`) — пайплайн из XLSX чеков в features
- **Кэширование** — parquet файлы для быстрой загрузки
- **Параллельная загрузка** XLSX через ThreadPoolExecutor

### 2. Модель
- Используется V3 модель (58 признаков, LightGBM)
- Модель обучена на спрос (demand), не на продажи
- Метрики: MAE ~2.9, WMAPE ~27%

### 3. Функционал

#### Прогнозирование
- `/predict` — прогноз для одного продукта на дату
- `/predict_all` — прогноз для всех продуктов пекарни на дату
- `/api/daily_plan` — почасовой план (6:00-22:00)

#### Типы дат
- **Исторические** — с данными о фактических продажах
- **Будущие** — прогноз на 14 дней вперёд

#### Погода
- Для исторических дат — из архива Open-Meteo
- Для будущих — прогноз Open-Meteo API

### 4. Frontend (`web/templates/index.html`)
- Выбор пекарни → продукта → даты
- Отображение прогноза, факта, ошибки
- График продаж за неделю (для исторических дат)
- Почасовой план (для всех дат)
- Фильтрация по категориям, поиск по продуктам

### 5. Проблемы и решения

#### Проблема: Только 1 дата в выборе
- В `web/data/` был только 1 XLSX файл (3 апреля)
- Решение: добавил исторические данные из `sales_hrs_all.csv` (30M строк)
- Лаги теперь берутся из истории

#### Проблема: Некорректные прогнозы (2 шт при факте 118)
- Причина: данные разных пекарень не объединялись
- "19 квартал 8/1 Казань Куюки" (история) vs "19 квартал 8/1 Казань Куюки (2)" (XLSX)
- Решение: нормализация названий пекарень (удаление "(2)", "(3)" и т.д.)

#### Проблема: Прогнозы на будущее
- Реализовано: генерация признаков на лету, прогноз погоды, сдвиг лагов

### 6. Не реализовано (нет данных)
- Остаток на конец дня — нет данных о производстве
- Упущенный спрос — нет данных о спросе vs продажах

## Запуск

```bash
# Установить зависимости
pip install -r requirements-dev.txt

# Запустить
python web/app.py
```

Откроется http://localhost:5000

## Структура файлов

```
web/
├── app.py              # Flask сервер
├── data_processing.py  # Обработка XLSX -> features
├── data/
│   ├── cache/          # Parquet кэш
│   ├── historical data/
│   │   └── sales_hrs_all.csv  # 30M чеков
│   └── *.xlsx          # Новые данные для прогноза
└── templates/
    └── index.html      # Frontend
```

## Данные
- XLSX: чеки с касс (текущие данные)
- Historical: 8 месяцев чеков для лагов
- Модель: V3 (demand, 58 признаков)
