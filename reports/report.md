# Отчёт: Прогнозирование спроса для сети пекарен «Бейгл»

## 1. Задача

Разработка модели машинного обучения для прогнозирования ежедневного спроса на продукцию по каждой пекарне сети «Бейгл» (Татарстан, Чувашия).

**Цель:** точный прогноз количества единиц продукции, которое будет продано в каждой пекарне на следующий день — для оптимизации выпуска и минимизации потерь (списания и упущенных продаж).

**Масштаб данных (v1 → v2):**

| | v1 (3 месяца) | v2 (8 месяцев) |
|---|---|---|
| Период | Янв–Мар 2026 | Авг 2025 – Апр 2026 |
| Строк | 322К | 3.5М |
| Пекарни | 173 | 199 |
| Продукты | 40 | 610 |
| Категории | 5 | 27 |
| Города | 7 | 9 (+ Курск, Москва) |

---

## 2. Что было сделано

### 2.1. Подготовка данных и признаков

Построен конвейер обработки данных, который из сырого Excel-файла продаж формирует набор из **35–52 признаков**:

| Группа признаков | Примеры | Зачем |
|-----------------|---------|-------|
| Продажи за прошлые дни | Вчера, 2 дня назад, неделю назад, 14 и 30 дней назад | Основа прогноза — модель учится на истории |
| Скользящие средние | Средние и разброс за 3, 7, 14, 30 дней | Улавливают тренд и стабильность спроса |
| Остатки и дефицит (v1) | Остаток вчера, соотношение остатка к продажам, дефицит | Учёт ситуаций, когда товар закончился раньше |
| Календарь | День недели, выходные, начало/конец месяца | Учёт недельной и месячной сезонности |
| Праздники и зарплата | Праздники, пред-/постпраздничные дни, неделя зарплаты | Всплески спроса в особые дни |
| Погода | Температура, осадки, ветер, холодная/плохая погода (15 признаков) | Влияние погоды на посещаемость пекарен |
| Лаги по спросу (v2) | demand_lag1..30, demand_roll_mean3..30 (11 признаков) | Прогнозирование истинного спроса, а не цензурированных продаж |

### 2.2. Модель

Использован алгоритм **LightGBM** (градиентный бустинг) — один из лучших алгоритмов для табличных данных. Гиперпараметры подобраны автоматически с помощью библиотеки Optuna (50 итераций подбора).

### 2.3. Эксперименты по улучшению (v1, 3 месяца)

Проведено **более 15 экспериментов** для повышения точности:

**Модельные эксперименты:**
| Эксперимент | Идея | Результат |
|-------------|------|-----------|
| Отдельные модели по категориям | Специализированная модель для каждой из 5 категорий | Хуже базовой (MAE +0.04) |
| Ансамбль моделей (Stacking) | Комбинация нескольких моделей | Хуже (MAE +0.26) |
| Кластеризация пекарен | Группировка похожих пекарен, модель на кластер | Хуже (MAE +0.10) |
| Двухэтапная модель | Сначала определить уровень спроса, потом точное число | Хуже (MAE +0.10) |
| Коррекция остатков | Дообучение на ошибках основной модели | Без эффекта |
| Асимметричная функция потерь | Штраф за недопрогноз больше, чем за перепрогноз | Снизила смещение (+0.9 → +0.15), точность не улучшила |

**Эксперименты с данными:**
| Эксперимент | Идея | Результат |
|-------------|------|-----------|
| Обрезка выбросов | Ограничить экстремальные продажи | Хуже — экстремальные продажи реальны |
| Фильтрация шума (медианное сглаживание) | Заменить скользящее среднее на медиану | **Небольшое улучшение (-0.009)** |
| Коррекция цензурированного спроса | Восстановить «истинный» спрос при стокауте | Хуже — модель уже учитывает остатки |
| Учёт упущенного спроса | Использовать данные об упущенных продажах (16 дней) | Данных слишком мало для эффекта |
| Анализ замещения | Рост продаж товара B при стокауте товара A | Эффект пренебрежимо мал (корреляция ~0.1) |

### 2.4. Эксперименты v2 (8 месяцев)

С получением расширенных данных (30М чеков с временем продажи за 8 месяцев) проведена **вторая серия экспериментов**:

**Базовые эксперименты:**
| # | Эксперимент | MAE | WMAPE | Delta vs baseline |
|---|-------------|-----|-------|-------------------|
| 01 | **Baseline 8m** | **2.29** | **25.9%** | — |
| 02 | Optuna 8m (20 триалов) | 2.28 | — | -0.01 |
| 03 | Demand target (спрос vs спрос) | 2.99* | 28.2%* | — |
| 05 | Product & bakery profiles | 2.31 | 26.1% | +0.02 |

\* Метрики exp 03 считаются по спросу, не по продажам — не сравнимы напрямую с baseline.

**High Demand Problem (эксперименты 40–46):**

Проблема: на товарах с продажами 100+ модель занижает прогноз (Bias +14, MAE 21–26). Причина — MSE-loss учит условное среднее, «усредняя» при широком распределении.

| # | Эксперимент | MAE | Delta | Суть |
|---|-------------|-----|-------|------|
| **43** | **Quantile P50 (медиана)** | **2.2655** | **-0.025** | Три модели: P25, P50, P75. Медиана устойчивее к выбросам |
| 40 | Tweedie | 2.2778 | -0.013 | Tweedie loss (power=1.5), учитывает гетероскедастичность |
| 45 | Log + Residual | 2.2800 | -0.010 | log1p(y) + коррекция остатков двумя моделями |
| 46 | Variance-weighted (1/std) | 2.2806 | -0.010 | Вес обратно пропорционален дисперсии продукта |
| 41 | Log-target | 2.2889 | -0.002 | Обучение на log1p(y), обратное преобразование |
| 44 | Mixture of Experts | 2.5371 | +0.247 | Раздельные модели high/low — потеря контекста |
| 42 | Asymmetric loss | 8.8391 | +6.55 | Кастомный gradient сломан, bias = +8.8 |

**Вывод:** лучший результат — Quantile P50 (MAE 2.27). Квантильная регрессия также даёт бизнес-ценность — интервалы прогноза [P25, P75] с покрытием 52%.

### 2.5. Прогнозирование спроса вместо продаж

**Проблема:** данные о продажах **цензурированы** — если товар закончился в 14:00, реальный спрос выше. 45% записей имеют признаки раннего стоп-аута (последняя продажа до 17:00). Предсказывать продажи = предсказывать «сколько продастся при текущем дефиците», а бизнесу нужен ответ «сколько выпустить».

**Метод восстановления спроса:**
1. Из 30М чеков с временем для каждого (товар × пекарня) строим кумулятивный часовой профиль продаж из «полных дней» (последняя продажа ≥ 17:00)
2. Для дней раннего стопа: `спрос = продано / cum_profile(час_стопа)`
3. Защиты: min 5 полных дней, cum ≥ 5%, cap ×5

**Честное сравнение (обе модели оценены по спросу):**

| | Model A (train на продажах) | Model B (train на спросе) |
|---|---|---|
| MAE vs спрос | 3.13 | **2.99** |
| WMAPE vs спрос | 29.5% | **28.2%** |
| Bias | +1.58 (занижает) | **+0.11 (точно)** |
| High demand (100+) Bias | +16.9 | **+14.4** |

Model B побеждает — обучение на спросе даёт более точные предсказания спроса.

### 2.6. Декомпозиция дисперсии

Исследование источников вариабельности на категории «Выпечка сытная» (std = 29.3):

| Уровень объяснения | Residual std | Доля объяснённая |
|-------------------|-------------|-----------------|
| Только день недели | 29.1 | 1.2% |
| + Продукт × день недели | 22.1 | 43.2% |
| + Пекарня × продукт × день | 10.7 | **86.7%** |

Модель получает все эти признаки (Пекарня, Номенклатура, ДеньНедели) и объясняет ~87% дисперсии. Оставшиеся ~13% — нередуцируемый шум (погода, случайный трафик и т. д.). Для топовых товаров (Треугольник курица, mean=90, residual_std=23) MAE ~15–20 — теоретический потолок.

### 2.7. Веб-интерфейс

Разработано веб-приложение (Flask) для удобного просмотра прогнозов: выбор пекарни, товара и даты с отображением прогнозируемого спроса.

## 3. Результаты

### 3.1. Метрики моделей

| Модель | Данные | MAE | WMAPE | Bias |
|--------|--------|-----|-------|------|
| v6 baseline | 3 мес, 5 категорий | 3.33 | 23.6% | +0.67 |
| **v2 baseline** | **8 мес, 27 категорий** | **2.29** | **25.9%** | **+0.08** |
| v2 Quantile P50 | 8 мес, 27 категорий | **2.27** | **25.6%** | +0.41 |
| v2 Demand model | 8 мес, спрос | 2.99* | 28.2%* | +0.11 |

\* Сравнение по спросу (не по продажам).

### 3.2. Экономический эффект

P&L за 7 дней, 173 пекарни, 5 категорий (3-месячные данные с реальным выпуском).
Параметры: маржа 60%, себестоимость 40%. Остаток не списывается сразу — 66% продаётся на следующий день как «вчерашний» со скидкой 30%.

| | Идеал | Пекарь (факт) | Model A (продажи) | Model B (спрос) |
|---|---|---|---|---|
| Выпуск | 788К шт | 704К шт | 655К шт | **834К шт** |
| Продано свежего | 788К шт | 605К шт | 586К шт | **681К шт** |
| Продано вчерашнего | 0 | 66К шт | 46К шт | **101К шт** |
| Чистое списание | 0 | 34К шт | 24К шт | 52К шт |
| Упущено | 0 | **183К шт** | 202К шт | **107К шт** |
| **Валовая прибыль** | **44.3 млн** | **33.4 млн** | **31.6 млн** | **38.2 млн** |
| **% от идеала** | 100% | 75.5% | 71.3% | **86.1%** |

**Экономия Model B vs пекарь: +673 тыс. руб/день (~20 млн руб/мес).**

Ключевой инсайт: пекарь угадывает общий объём выпуска неплохо (704К при спросе 788К), но **неправильно распределяет ассортимент** — одних товаров перебор, других недобор. Модель точнее предсказывает ассортиментную разбивку, поэтому упущенных продаж в 1.7 раза меньше (107К vs 183К).

Данные об остатках или списаниях **не требуются для работы модели** — она опирается только на историю продаж, календарь и погоду. Задержки в получении данных по списаниям не влияют на качество прогноза.

## 4. Ключевые выводы

1. **Модель работает и даёт практически применимые прогнозы.** Средняя ошибка 2.29 единицы на 8 месяцах данных (против 3.33 на 3 месяцах). Расширение данных дало -31% MAE.

2. **Прогноз спроса ценнее прогноза продаж.** Модель, обученная на восстановленном спросе (Model B), приносит на 673 тыс. руб/день больше, чем текущее решение пекаря — за счёт более точного ассортиментного распределения.

3. **45% данных цензурированы.** Почти половина записей имеют признаки раннего стоп-аута. Обучение на «сырых» продажах систематически занижает прогноз. Кумулятивные профили из 30М чеков решают эту проблему.

4. **Единая модель лучше специализированных.** Все попытки разделить модель по категориям, уровням спроса (MoE) или кластерам ухудшали результат.

5. **Квантильная регрессия** (P25/P50/P75) — лучший подход: MAE 2.27 (P50) + интервалы прогноза с покрытием 52%. Даёт пекарю не одно число, а диапазон.

6. **Потолок точности.** ~87% дисперсии объясняется комбинацией (пекарня × продукт × день недели). Оставшиеся ~13% — непредсказуемый шум. Для хитовых товаров (Треугольник курица, mean=90) MAE ~15–20 — фундаментальный предел.

7. **Остаток не сгорает.** 8.5% всех продаж — вчерашний товар со скидкой ~30%. Это делает стратегию «лучше выпустить больше» экономически оправданной — остаток частично возвращается через допродажу.
