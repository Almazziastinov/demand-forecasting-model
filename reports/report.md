# Отчёт: Прогнозирование спроса для сети пекарен «Бейгл»

## 1. Задача

Разработка модели машинного обучения для прогнозирования ежедневного спроса на продукцию по каждой пекарне сети «Бейгл» (Татарстан, Чувашия).

**Цель:** точный прогноз количества единиц продукции, которое будет продано в каждой пекарне на следующий день — для оптимизации выпуска и минимизации потерь (списания и упущенных продаж).

**Масштаб данных:**
- 3 месяца исторических продаж
- 7 городов (Казань, Чебоксары, Набережные Челны, Нижнекамск, Зеленодольск, Бугульма, Заинск)
- 173 пекарни
- 40 наименований продукции в 5 категориях

---

## 2. Что было сделано

### 2.1. Подготовка данных и признаков

Построен конвейер обработки данных, который из сырого Excel-файла продаж формирует набор из **35 признаков**:

| Группа признаков | Примеры | Зачем |
|-----------------|---------|-------|
| Продажи за прошлые дни | Вчера, 2 дня назад, неделю назад, 14 и 30 дней назад | Основа прогноза — модель учится на истории |
| Скользящие средние | Средние и разброс за 3, 7, 14, 30 дней | Улавливают тренд и стабильность спроса |
| Остатки и дефицит | Остаток вчера, соотношение остатка к продажам, дефицит | Учёт ситуаций, когда товар закончился раньше |
| Календарь | День недели, выходные, начало/конец месяца | Учёт недельной и месячной сезонности |
| Праздники и зарплата | Праздники, пред-/постпраздничные дни, неделя зарплаты | Всплески спроса в особые дни |
| Погода | Температура, осадки, холодная/плохая погода | Влияние погоды на посещаемость пекарен |

### 2.2. Модель

Использован алгоритм **LightGBM** (градиентный бустинг) — один из лучших алгоритмов для табличных данных. Гиперпараметры подобраны автоматически с помощью библиотеки Optuna (50 итераций подбора).

### 2.3. Эксперименты по улучшению

Проведено **более 15 экспериментов** для повышения точности:

**Модельные эксперименты:**
| Эксперимент | Идея | Результат |
|-------------|------|-----------|
| Отдельные модели по категориям | Специализированная модель для каждой из 5 категорий | Хуже базовой (MAE +0.04) |
| Ансамбль моделей (Stacking) | Комбинация нескольких моделей | Хуже (MAE +0.26) |
| Кластеризация пекарен | Группировка похожих пекарен, модель на кластер | Хуже (MAE +0.10) |
| Двухэтапная модель | Сначала определить уровень спроса, потом точное число | Хуже (MAE +0.10) |
| Коррекция остатков | Дообучение на ошибках основной модели | Без эффекта |
| Асимметричная функция потерь | Штраф за недопрогноз больше, чем за перепрогноз | Снизила смещение (+0.9 → +0.15), точность не улучшила |

**Эксперименты с данными:**
| Эксперимент | Идея | Результат |
|-------------|------|-----------|
| Обрезка выбросов | Ограничить экстремальные продажи | Хуже — экстремальные продажи реальны |
| Фильтрация шума (медианное сглаживание) | Заменить скользящее среднее на медиану | **Небольшое улучшение (-0.009)** |
| Коррекция цензурированного спроса | Восстановить «истинный» спрос при стокауте | Хуже — модель уже учитывает остатки |
| Учёт упущенного спроса | Использовать данные об упущенных продажах (16 дней) | Данных слишком мало для эффекта |
| Анализ замещения | Рост продаж товара B при стокауте товара A | Эффект пренебрежимо мал (корреляция ~0.1) |

### 2.4. Веб-интерфейс

Разработано веб-приложение (Flask) для удобного просмотра прогнозов: выбор пекарни, товара и даты с отображением прогнозируемого спроса.

## 3. Результаты

### Итоговые метрики лучшей модели (v6)

| Метрика | Значение | Что означает |
|---------|----------|-------------|
| **MAE** | **3.33** | В среднем прогноз отклоняется от реальных продаж на 3.33 единицы |
| **WMAPE** | **23.6%** | Взвешенная средняя процентная ошибка |
| **Bias** | **+0.67** | Модель в среднем немного завышает прогноз (лучше, чем занижать) |

### Точность по категориям

| Категория | Средний спрос | MAE | Ошибка относительно спроса |
|-----------|:------------:|:---:|:--------------------------:|
| Выпечка сытная | 24.5 шт | 5.2 | ~21% |
| Фастфуд | 10.6 шт | 3.0 | ~28% |
| Выпечка сладкая | 9.3 шт | 2.6 | ~28% |
| Пироги сытные | 2.4 шт | 1.0 | ~42% |
| Пироги сладкие | 2.1 шт | 0.9 | ~44% |

Наиболее точный прогноз — для категории «Выпечка сытная» (21% ошибки при самом высоком спросе). Для пирогов ошибка в процентах выше, но в абсолютных единицах составляет менее 1 штуки.

## 4. Ключевые выводы

1. **Модель работает и даёт практически применимые прогнозы.** Средняя ошибка 3.3 единицы при среднем спросе ~14 единиц.

2. **Имеющиеся данные не дают полное представление спроса.** Данные основаны на продажах не дают точного представления о спросе, тк выпуск не является независимой величиной и исходит из (локальных) продаж, соответсвенно разница между выпуском и продажами - остаток вместе с продажами не дает нам представления реального спроса.

3. **Сложность получения более детализированных данных.** Решением проблемы спроса является детализация данных, если мы знаем время последенго чека и потенциальную выгоду то мы можем вывести приблизительное значение спроса, но тк детализированные данные имеют существенно больший объем, на данном этапе нет возможности для получения.

4. **Единая модель лучше, чем набор специализированных.** Все попытки разделить модель по категориям, кластерам или уровням спроса ухудшали результат — глобальная модель лучше обобщает закономерности.

5. **Погода — полезный фактор.** Температура вошла в топ-5 важных признаков модели.

6. **Данные об остатках дают значимый эффект, но...** В текущей системе данные о продажах и остатки не отображают реальную картину спроса.

7. **Потолок точности при текущих данных достигнут.** MAE ~3.33 — лучший результат, который стабильно воспроизводится. 15+ экспериментов не дали значимого улучшения.

## 5. Рекомендации по дальнейшему улучшению

Для значимого повышения точности прогноза рекомендуется:

| Направление | Ожидаемый эффект | Сложность |
|-------------|-----------------|-----------|
| **Увеличить историю до 6-12 месяцев** | Модель сможет учитывать годовую сезонность | Низкая — нужно только накопить данные |
| **Добавить почасовые данные продаж** | Понимание внутридневных пиков | Средняя — нужна доработка учётной системы |
| **Данные о промоакциях и маркетинге** | Учёт запланированных всплесков спроса | Средняя — нужна интеграция с маркетингом |
| **Данные о трафике / событиях в городе** | Учёт внешних факторов посещаемости | Высокая — внешние источники данных |
